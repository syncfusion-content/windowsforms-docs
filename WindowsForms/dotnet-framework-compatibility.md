# .NET Framework and .NET Core Compatibility

Syncfusion<sup>®</sup> Windows Forms Controls are compatible with the latest supported .NET Framework and .NET (Core) runtimes. Use the compatibility matrix below to identify whether your installed Syncfusion<sup>®</sup> Essential Studio<sup>®</sup> version supports the target .NET runtime.

## Overview

Use this page to:

1. Identify the Syncfusion<sup>®</sup> Essential Studio<sup>®</sup> version installed in your project.
2. Locate the row in the compatibility matrix that corresponds to that version (or the nearest lower version).
3. Read across the row to confirm support for the .NET runtime you plan to target.
4. If your Syncfusion<sup>®</sup> Essential Studio<sup>®</sup> version does not list the required runtime, upgrade to a supported Syncfusion<sup>®</sup> release or contact Syncfusion<sup>®</sup> support for guidance on legacy versions.

> Syncfusion<sup>®</sup> uses `.NET Core` to refer to .NET Core 1.x–3.x and uses `.NET` to refer to .NET 5.0 and later. This document uses the same convention. Refer to the corresponding version release notes for detailed runtime support information.

## Prerequisites

The following prerequisites apply when targeting the runtimes listed in the compatibility matrix:

* A supported Syncfusion<sup>®</sup> Essential Studio<sup>®</sup> Windows Forms installation that matches the version shown in the matrix.
* Visual Studio with the appropriate workload installed for the target .NET runtime. For .NET 6.0 and later, install the matching .NET SDK.
* The required .NET Framework and/or .NET runtimes installed on the development and target machines.
* Syncfusion<sup>®</sup> Windows Forms NuGet packages or assemblies referenced in your project.

Support may vary depending on the installed Syncfusion<sup>®</sup> version. Verify the exact prerequisites for your version in the [Installation](https://help.syncfusion.com/windowsforms/installation) documentation.

### Legend

The values in the compatibility matrix are defined as follows:

* **Yes** — The Syncfusion<sup>®</sup> Essential Studio<sup>®</sup> version supports the corresponding .NET runtime.
* **No** — The Syncfusion<sup>®</sup> Essential Studio<sup>®</sup> version does not support the corresponding .NET runtime.
* **N/A** — The .NET runtime had not yet been released at the time of the corresponding Syncfusion<sup>®</sup> version. Support is not applicable.
* **SP** — Service Pack release of the corresponding Syncfusion<sup>®</sup> Essential Studio<sup>®</sup> volume.

The `.NET 4.6.2 & above` column represents all .NET Framework versions from 4.6.2 onward, including 4.7, 4.7.1, 4.7.2, and 4.8.x. Refer to the Syncfusion<sup>®</sup> release notes for the latest compatibility information.

### Unsupported Versions

If your Syncfusion<sup>®</sup> Essential Studio<sup>®</sup> version does not support the target .NET runtime, consider the following options:

* Upgrade to a supported Syncfusion<sup>®</sup> Essential Studio<sup>®</sup> version.
* Review the [Release Notes](https://www.syncfusion.com/release-notes/windows-forms) for the version that introduced support for the target runtime.
* Contact Syncfusion<sup>®</sup> support for guidance on legacy versions.

Support may vary depending on the installed Syncfusion<sup>®</sup> version.

## Troubleshooting

The following table lists common issues that may occur when targeting the runtimes shown in the compatibility matrix.

<table>
<tr>
<th>Issue</th>
<th>Possible cause</th>
<th>Suggested action</th>
</tr>
<tr>
<td>NuGet restore fails for Syncfusion<sup>®</sup> packages</td>
<td>Project targets a .NET runtime that is not supported by the installed Syncfusion<sup>®</sup> version</td>
<td>Refer to the compatibility matrix and target a supported runtime, or upgrade the Syncfusion<sup>®</sup> version.</td>
</tr>
<tr>
<td><code>System.IO.FileNotFoundException</code> at application startup</td>
<td>Syncfusion<sup>®</sup> assemblies built for a different .NET runtime are referenced</td>
<td>Reinstall the Syncfusion<sup>®</sup> assemblies that match the target .NET runtime.</td>
</tr>
<tr>
<td>Windows Forms designer fails to load a Syncfusion<sup>®</sup> control</td>
<td>Designer host targets a .NET runtime that is not supported by the installed Syncfusion<sup>®</sup> version</td>
<td>Retarget the designer host to a supported .NET runtime or upgrade the Syncfusion<sup>®</sup> version.</td>
</tr>
<tr>
<td>Assembly resolution errors at build or run time</td>
<td>Conflicting Syncfusion<sup>®</sup> assemblies from multiple versions are referenced</td>
<td>Remove older Syncfusion<sup>®</sup> assembly references and re-add the version that matches the target runtime.</td>
</tr>
</table>

If the issue persists, refer to the Syncfusion<sup>®</sup> release notes for the installed version or contact Syncfusion<sup>®</sup> support.
